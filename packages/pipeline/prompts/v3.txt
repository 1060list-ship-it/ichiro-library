## 絶対制約（歌詞引用禁止）
- 出力のどの項目でも、歌詞フレーズを引用・生成・要約・言い換えしないこと。字幕が断片的でも補完しないこと。
- 楽曲について触れる場合は、曲名と行為だけを書くこと。許容されるのは「〇〇という曲を歌った」「〇〇を演奏した」「〇〇について話した」のような事実ベースの記述のみ。
- 「歌詞に込められた意味」「歌詞の一節」「〇〇という言葉」「サビで歌っていた内容」のような歌詞内容への言及は禁止。
- 字幕に歌詞らしき断片があっても、songs や has_live_singing の判定材料にだけ使い、文章としては一切出力しないこと。
- 曲名を出力する場合は、必ず後述の songs テーブル正式表記一覧にある表記を使うこと。

### 1. summary（概要）
この配信のライブレポートとして500〜800字で書いてください。

**書き方の指針：**
- 「この夜の一郎は〜」「配信の前半は〜、後半に入ると〜」のように、配信の流れを追いながら読み物として書く
- 山口一郎の語り口・その日の雰囲気・印象的な発言や場面を盛り込む
- 配信を見ていない読者が「どんな夜だったか」臨場感を持って理解できる粒度
- 箇条書き・見出しは使わない。流れるような文章（段落分け可）で書く
- ゲストがいる場合は会話のトーンや化学反応にも触れる
- 曲に触れる場合は、正式曲名と出来事だけを書くこと。歌詞内容や歌詞の解釈は書かないこと。
- 歌唱のみ・楽器演奏のみの配信は、セットリストと全体の雰囲気を中心に 200〜400字 でまとめること（無理に 500〜800字 に引き延ばさない）。

### 2. chapters（チャプター）
配信の中で**明確な話題転換があった場合のみ**生成してください。チャプターはファンが「どこから見ればいいか」を判断するための目次です。
各チャプターに以下を含めてください:
- start_sec: 開始秒数（整数）。**必ず字幕の `[hh:mm:ss]` タイムスタンプを秒数（時×3600+分×60+秒）に変換した値を使うこと。** 話題が実際に切り替わる字幕行のタイムスタンプを使い、前チャプターの end_sec と同じ値にしてはいけない。
- end_sec: 終了秒数（整数）。次チャプターの start_sec と同じ値。
- title: チャプタータイトル（20字以内）
- summary: そのチャプターの要約（50〜100字）

**生成ルール：**
- 「配信開始」「オープニング」「はじまり」など 00:00 付近の自明なチャプターは作らないこと。
- 1つの話題ブロックは 15〜20 分以上が目安。細切れにしないこと。
- 歌唱のみ・楽器演奏のみで話題転換がない配信は空配列 `[]` を返すこと。
- 生成すべき明確な転換点がない場合は無理に作らず `[]` を返すこと。

チャプター summary でも歌詞内容には触れないこと。曲の話題を書く場合は正式曲名と出来事だけを書くこと。

### 3. corner_names（コーナー名）
字幕テキストに**明示的に名前が登場した**コーナーのみ抽出してください。
推測・補完は不可。言及がなければ空配列 [] を返してください。
1つの配信に複数のコーナーが混在することがあります。すべて抽出してください。
コーナーが登場しない配信もあります。その場合は空配列 [] を返してください。

既知のコーナー名（これ以外のコーナー名も存在します）:
- 未知との遭遇（電話・通話でゲストと話すコーナー。UFO・霊体験・初体験など「普通ではない特殊な体験談」をテーマにしているのが特徴）
- 深夜対談（必ず電話・通話で深夜に対談するコーナー。通話相手が登場した深夜配信の場合はこのコーナーと判断してよい）
- ライブビデオ解説（過去のライブ映像を視聴・鑑賞するシーン全般。裏話・解説がなく黙って視聴するだけの場面も含む）
- ゲーム実況（ゲームをプレイしながら配信）
- NF（ノーフィルター）
- 音楽の話
- リスナー相談

**補正ルール：コーナーとして明示的にアナウンス・紹介されている場合に限り、字幕の表記がリストのコーナー名の誤変換と判断できるときは正式名称を使用すること。ただし会話中に偶然似た言葉が出ただけの場合はコーナーとして抽出しないこと。**

### 4. guests（ゲスト名）
配信に**実際に参加・出演した**ゲストのフルネームのみ抽出してください。
スタジオに来たゲストだけでなく、**電話・通話で参加した人物も含めてください**。
山口一郎本人・リスナーのニックネーム・コメントで言及されただけの人名は含めないでください。
ゲストがいなければ空配列 [] を返してください。

### 5. tags（タグ）
以下の制御語彙リストから、この配信に該当するタグを **2〜5個** 選んで返してください。
リスト外のタグは生成しないこと。汎用的すぎて毎回付くタグ（雑談・ファン交流・人間関係）は除外済み。

【配信スタイル】
人生相談 / 恋愛相談

【音楽・制作】
音楽制作 / 制作秘話 / 楽曲解説 / ライブレポート / ライブ演出裏話 / フェス / ツアー / 音楽業界 / ラジオ / オールナイトニッポン

【ゲーム】
ゲーム

【テーマ・話題】
メンタルヘルス / うつ病 / AI / 哲学 / 社会問題 / ファッション / 野球 / サウナ / グッズ紹介

【特別コンテンツ】
ゲスト / コラボ / 新曲発表

### 6. songs（登場した楽曲名）
配信中に**実際に流れた・演奏された・解説された**楽曲名を抽出してください。
単に会話で言及されただけの曲は含めないでください（解説・視聴・演奏された曲のみ）。
楽曲がなければ空配列 [] を返してください。
特に「ライブビデオ解説」コーナーでは曲名を漏れなく抽出してください。
**注意：ツアー名・イベント名・コーナー名は含めないこと。楽曲名とツアー名が同じ単語の場合（例：「怪獣」は曲名でもありツアー名でもある）は、文脈から「実際に演奏・再生された曲」かどうかを判断すること。**
**表記ルール：songs に入れる曲名は、必ず後述の songs テーブル正式表記一覧と完全一致させること。**

### 7. has_live_singing（生歌唱の有無）
配信中に山口一郎が**実際にその場で歌を歌った**シーンがあればtrue、なければfalseを返してください。
ライブビデオの再生・音源の流し聞きは含めません。その場で歌った場合のみtrueです。
**重要：配信の冒頭や転換時に自身の楽曲がBGMとして流れるパターンがある。字幕に歌詞らしき断片が出ていても、それが冒頭数分以内かつトーク発言と混在していない場合はBGM再生とみなし、falseとすること。**

### 8. talk_topics（対談・トークのテーマ）
「深夜対談」「未知との遭遇」などのトークコーナーで話し合われた主なテーマを抽出してください。
例: ["AI論", "音楽制作の裏側", "恋愛観"]
対談・トークがなければ空配列 [] を返してください。

### 9. highlights（盛り上がりの瞬間）
配信の中で「これだけ見れば雰囲気がわかる」と感じる、印象的・面白い・感動的な瞬間を **0〜5個** 抽出してください。
各highlightに以下を含めてください:
- start_sec: 開始秒数（整数）。**必ず字幕の `[hh:mm:ss]` タイムスタンプを秒数（時×3600+分×60+秒）に変換した値を使うこと。** quote として引用する発言が字幕に登場する行のタイムスタンプを使う。
- quote: その瞬間の発言の引用（50字以内）
- reason: 盛り上がりの種別（「笑い」「名言」「感動」「驚き」のいずれか）

**生成ルール：**
- 字幕から直接引用できる発言を優先すること。
- quote には歌詞フレーズを絶対に入れないこと。歌唱シーンが盛り上がりでも、前後のMCや説明など非歌詞の発言を引用すること。非歌詞の引用が取れない場合は、その場面を highlight に選ばないこと。
- 歌唱のみ・楽器演奏のみで印象的なトーク発言がない配信は空配列 `[]` を返すこと。
- 無理に個数を埋めないこと。本当に印象的な瞬間だけを選ぶこと。印象的な瞬間がなければ `[]` でよい。
